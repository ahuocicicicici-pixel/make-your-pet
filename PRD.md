# 雪纳瑞桌宠 PRD(v1.0 规划)

| 项目 | 内容 |
|---|---|
| 产品名 | Schnauzer Pet(雪纳瑞桌宠) |
| 平台 | macOS(Electron),后续可扩展 Windows |
| 当前版本 | v0.1.0(原型:贴图动画 + 悬浮按钮) |
| 目标版本 | v1.0(完整养成型桌宠) |

---

## 1. 背景与问题

v0.1 原型验证了"透明置顶窗口 + 精灵图动画"的技术路线可行,但距离"产品"有三个核心差距:

| # | 问题 | 根因 | 解法方向 |
|---|---|---|---|
| P1 | 画面模糊 | 每帧素材仅 192×208 物理像素,在 Retina(2x)屏上被拉伸到约 454×490 物理像素显示,欠采样 ~2.4 倍 | 素材全部按 @2x 重制(见 §6.1) |
| P2 | 互动方式反直觉 | 现在靠悬浮按钮触发动作,桌宠的天然交互应是"直接点宠物" | 部位点击 + 手势交互(见 §4) |
| P3 | 没有玩法,留不住人 | 只有动画播放,无状态、无目标、无反馈循环 | 引入养成系统:状态条 / 喂食 / 打工 / 商店 / 洗澡 / 娱乐(见 §5) |

## 2. 产品定位

**一只活在 Mac 桌面角落、需要被照顾、也会陪你工作的雪纳瑞。**

- 对标参考:QQ 宠物(养成数值循环)、Shimeji(桌面行为)、Stray Cat / Bongo Cat(陪伴感)。
- 核心情感:**陪伴 + 轻度责任感**。状态会衰减 → 用户产生"该喂它了"的牵挂;打工赚钱 → 形成"付出-回报"循环。
- 反目标(明确不做):不做重度游戏(战斗/PVP/社交)、不做强打扰(不弹强制窗口、不阻塞用户工作)、不做内购。

### 目标用户

- 主力:学生 / 上班族,电脑前久坐,喜欢萌系治愈内容;
- 场景:写代码/写文档时瞥一眼,休息间隙逗一逗,一天交互 5–20 次、每次 <30 秒。

## 4. 交互设计

### 4.1 部位点击(P2 核心)

宠物站立姿态划分 6 个热区(hitbox 按帧内相对坐标定义,随缩放等比换算):

```
        ┌─────────┐
        │   头部   │  ← 摸头
        │ ┌─────┐ │
        │ │ 脸  │ │  ← 亲昵
        └─┤     ├─┘
   ┌──────┴─────┴──────┐
   │       背部         │ ← 顺毛
   │  ┌─────────┐      │
   │  │  肚子    │ 尾巴 │ ← 挠痒 / 转圈
   │  └─────────┘      │
   │   爪子(前肢)       │ ← 握手
   └───────────────────┘
```

| 部位 | 单击反应 | 数值影响 | 动画 |
|---|---|---|---|
| 头部 | 摸头:眯眼蹭手 | 心情 +2 | `pet-head` |
| 脸部 | 亲昵:伸舌头舔 | 心情 +3 | `lick` |
| 肚子 | 挠痒:翻肚皮蹬腿 | 心情 +3 | `belly-rub` |
| 爪子 | 握手:抬爪 | 心情 +1 | `shake-paw` |
| 尾巴 | 原地转圈追尾巴 | 心情 +1 | `chase-tail` |
| 背部 | 顺毛:舒服地抖一抖 | 心情 +2 | `shiver-happy` |

防刷规则:同一部位 60 秒内重复点击,第 3 次起不再加心情,宠物播放"不耐烦"动画(扭头);连续点击 10+ 次播放"生气"动画并走开(负反馈也是萌点)。

### 4.2 其他手势

| 手势 | 行为 |
|---|---|
| 单击(身体任意处,非热区) | 抬头看向用户,随机吠一声(气泡"汪!") |
| 双击 | 呼出/收起 **互动面板**(见 §4.3) |
| 拖拽 | 拎起(四肢下垂的"被拎"动画)→ 松手后落地抖毛 |
| 拖到屏幕边缘 | 吸附在边缘坐下 |
| 右键 | 系统菜单:面板 / 商店 / 设置 / 退出 |
| 长时间无操作 | 自主行为:随机漫步、趴下、睡觉、对镜头卖萌(随机间隔 1–5 分钟) |

### 4.3 互动面板(双击呼出)

挂在宠物旁边的小卡片(非独立大窗口),包含:

```
┌──────────────────────────┐
│ 🦴 饱食度 ████████░░ 80   │
│ 🛁 清洁度 ██████░░░░ 60   │
│ 😊 心情值 █████████░ 90   │
│ ❤️ 健康值 ██████████ 100  │
│ ────────────────────────  │
│ 💰 327   Lv.5  ⭐ 320/500 │
│ ────────────────────────  │
│ [喂食] [洗澡] [玩耍]       │
│ [打工] [商店] [背包]       │
└──────────────────────────┘
```

- 状态条低于 30 时对应行变红闪烁;
- 面板 8 秒无操作自动收起;
- 菜单栏托盘图标同步显示最低的一项状态(如 `🦴 23`),作为不打扰的提醒入口。

## 5. 养成系统(P3 核心)

### 5.1 四条状态 + 衰减

所有状态范围 0–100,**离线也按时间结算**(重新打开时按离线时长补算,但衰减减半,避免惩罚感过强)。

| 状态 | 衰减速率(在线) | 提升手段 | 跌到 0 的后果 |
|---|---|---|---|
| 饱食度 | -4 / 小时 | 喂食 | 心情衰减 ×2,播放"饿瘫"动画 |
| 清洁度 | -2 / 小时;打工额外 -10/次;玩耍 -5/次 | 洗澡 | 身上出现污渍贴图,心情衰减 ×1.5 |
| 心情值 | -3 / 小时;受饱食/清洁拖累加速 | 抚摸、玩耍、喂零食 | 拒绝打工,趴着不理人 |
| 健康值 | 不自然衰减;饱食或清洁 =0 持续 6h 后开始 -2/h | 用背包"药品";其余三项均 >50 时 +5/h 自动回复 | 生病动画,所有互动收益减半 |

> 数值设计原则:一个普通用户**每天照顾 2–3 次、每次 1 分钟**即可把所有状态维持在 70+。衰减不是惩罚,是制造"被需要"的钩子。

### 5.2 喂食

- 入口:面板 [喂食] → 弹出背包食物栏;或从背包拖拽食物到宠物嘴边(拖喂,优先实现面板点喂)。
- 流程:选择食物 → 食物落地 → 宠物跑过去 → 吃饭动画(按食物大小 3–8 秒)→ 数值结算 → 满足气泡(❤️)。
- 饱食度 ≥95 时拒食(扭头),防止数值溢出刷经验。

### 5.3 打工(金币来源)

QQ 宠物式的核心循环:**打工赚钱 → 商店买吃的/玩具 → 养得更好**。

| 工作 | 时长 | 报酬 | 解锁等级 | 要求 |
|---|---|---|---|---|
| 送报纸 | 15 分钟 | 💰20 + 10exp | Lv.1 | 心情 ≥30 |
| 看仓库 | 30 分钟 | 💰50 + 25exp | Lv.3 | 心情 ≥30,健康 ≥50 |
| 导盲犬实习 | 60 分钟 | 💰120 + 60exp | Lv.6 | 全状态 ≥50 |
| 警犬训练营 | 120 分钟 | 💰300 + 150exp | Lv.10 | 全状态 ≥60 |

- 打工期间宠物**从桌面消失**(留一个小牌子"打工中 23:45 ⏱",点击可提前召回、报酬减半);
- 结束时系统通知 +宠物跑回来的动画 + 金币入账音效;
- 打工返回后清洁度 -10、饱食度 -10:**赚了钱就要花**,闭环成立。

### 5.4 商店与背包

商店三个 tab(价格示例,可调):

| 类别 | 商品示例 | 价格 | 效果 |
|---|---|---|---|
| 食物 | 狗粮 | 💰10 | 饱食 +20 |
| | 罐头 | 💰25 | 饱食 +40,心情 +5 |
| | 大骨头 | 💰50 | 饱食 +70,心情 +10 |
| | 小饼干(零食) | 💰8 | 饱食 +5,心情 +8 |
| 日用 | 洗澡香波 | 💰15 | 洗澡必需消耗品,清洁 +60 |
| | 感冒药 | 💰40 | 健康 +50 |
| 玩具 | 网球 | 💰30 | 解锁"扔球"玩耍(永久) |
| | 飞盘 | 💰80 | 解锁"接飞盘"玩耍(永久) |
| 装饰 | 红色围巾 | 💰200 | 外观佩戴(永久,叠加渲染层) |

新手引导:首次启动赠送 💰50 + 狗粮 ×3 + 香波 ×1,确保第一天不用打工也能完整体验一轮"喂-洗-玩"。

### 5.5 洗澡与娱乐

- **洗澡**:消耗香波 → 浴缸 + 泡泡动画(8 秒)→ 清洁 +60、心情 +5。清洁度 <40 时宠物身上出现可见污渍层,洗完消失(视觉反馈直接驱动行为)。
- **玩耍(扔球)**:点 [玩耍] → 鼠标变成球 → 在桌面任意处点击落球 → 宠物跑过去叼回原位 → 心情 +10、清洁 -5。每天前 5 次有收益,之后纯互动。

### 5.6 等级(长期目标)

- 经验来源:喂食 +5、洗澡 +8、玩耍 +5、打工(见表)。
- 升级曲线:`下一级所需 = 100 × 1.4^(lv-1)`,Lv.10 约需累计两周日常。
- 等级用途:解锁更高级打工、商店进阶商品、特殊动画(Lv.5 解锁"睡觉吐泡泡"等彩蛋)。

### 5.7 一日节律(自主行为)

| 时段(跟随系统时间) | 行为 |
|---|---|
| 07:00–09:00 | 早晨拉伸、精神活跃,心情自然 +5 |
| 12:00–13:00 | 如果饱食 <60,坐着盯用户看 + "肚子咕咕叫"气泡 |
| 22:30 以后 | 自动趴窝睡觉(呼吸起伏 + Zzz),衰减全部减半;点击会被叫醒(心情 -2,劝你别闹) |

## 6. 技术方案

### 6.1 高分辨率素材规范(解决 P1)

| 项 | 现状 | v1.0 要求 |
|---|---|---|
| 单帧尺寸 | 192×208 物理像素 | **512×512 物理像素**(@2x,逻辑显示 ≤256pt) |
| 组织方式 | 单张大精灵图(1536×1872) | 按动作拆分目录 `assets/anim/<action>/frame_%02d.png`,打包时合成 @1x/@2x 两套精灵图 |
| 格式 | webp 有损 | PNG(制作)→ webp 无损(分发) |
| CSS | `background-position` 切帧 | 不变,但 `background-size` 按 DPR 选 @1x/@2x;`image-rendering: auto` |

动作清单(v1.0 共 22 组,新增 14 组):

```
已有重制:idle / run-left / run-right / wave / jump / fail / wait / work / review
新增:pet-head / lick / belly-rub / shake-paw / chase-tail / shiver-happy /
      eat / bath / sleep / sick / dirty-overlay / carried(被拎)/ angry / bark
```

素材生产方式:沿用现有 AI 生成卡通风格(与 `teacher-*.png` 同风格),按 512×512 透明底逐帧出图,人工筛选后入库。

### 6.2 点击命中与窗口策略(解决 P2)

- 窗口保持 `transparent + frame:false`,**移除全局 `-webkit-app-region: drag`**(它会吃掉 click 事件),改为:
  - 渲染层 `mousedown + mousemove` 自行实现拖拽(超过 4px 位移判定为拖,否则判定为点击);
  - 点击坐标 → 当前帧的部位 hitbox 表(JSON,按动作+帧定义矩形区域)→ 分发动作。
- `setIgnoreMouseEvents(true, {forward:true})` + 宠物身体区域动态开启鼠标接收:窗口空白区域**点击穿透**到桌面(当前 300×320 的整个窗口都会挡鼠标,是隐性 bug,v1.0 必须修)。

### 6.3 架构与数据

```
main process
├── 窗口管理(宠物窗 / 面板窗 / 商店窗)
├── 状态引擎(唯一数据源,1 次/分钟 tick 结算衰减)
├── 持久化:electron-store → ~/Library/Application Support/Schnauzer Pet/save.json
├── 打工定时器(主进程持有,渲染层只显示倒计时)
└── 托盘 + 系统通知

renderer(宠物窗)
├── 动画状态机(XState 或手写 FSM:状态 × 事件 → 动作)
├── hitbox 命中 → IPC 上报 main
└── 气泡/污渍/装饰叠加层

IPC 约定:渲染层永不直接改数值,全部走 `state:action` 事件 → main 结算 → 广播 `state:updated`
```

存档结构(示意):

```json
{
  "version": 1,
  "stats": { "hunger": 80, "clean": 60, "mood": 90, "health": 100 },
  "coins": 327, "level": 5, "exp": 320,
  "inventory": [{ "id": "dogfood", "count": 3 }],
  "unlocked": ["ball"], "wearing": [],
  "work": { "jobId": null, "endsAt": null },
  "lastTickAt": "2026-06-10T13:00:00+08:00"
}
```

### 6.4 非功能性要求

| 项 | 指标 |
|---|---|
| 内存 | 常驻 ≤180MB(动画帧懒加载,非当前动作的图集不驻留) |
| CPU | 待机 ≤2%(idle 动画 4fps;窗口失焦不降帧——桌宠本来就在后台) |
| 包体 | DMG ≤150MB |
| 数据安全 | 存档写入采用临时文件+原子改名,崩溃不丢档 |
| 打扰度 | 系统通知每天 ≤3 条(打工完成 / 状态 <20 各一类,可在设置中关闭) |

## 8. 关键风险

| 风险 | 影响 | 缓解 |
|---|---|---|
| AI 生成 22 组动作风格不一致 | 观感廉价 | 锁定同一参考图 + 提示词模板;先小批量验收 idle/eat 两组定风格基线 |
