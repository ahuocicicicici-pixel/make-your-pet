# 给 Codex 的任务说明（从这里开始读）

你好 👋 你的任务是**用 image2 为这只雪纳瑞桌宠生成全部美术素材**（57 个动作的逐帧 PNG）。
代码框架已经搭好，缺的只有图。你**只需要产出图片文件**，放到指定目录即可，App 会自动用上。

---

## 一、先读这几个文件（按顺序）

| 顺序 | 文件 | 你要从中获取 |
|---|---|---|
| 1 | [`references/README.md`](references/README.md) | 7 张造型参考图，以及"哪张图配哪些动作" |
| 2 | [`ART-PRD.md`](ART-PRD.md) | **核心作业书**。§0 交付契约、§2 角色圣经、§3 生成规范、§6 动作总表、§7 逐帧提示词 |
| 3 | [`assets/anim/manifest.json`](assets/anim/manifest.json) | **唯一真相源**：每个动作的 id、帧数、帧率、循环。以它为准，别改 |

产品背景（可选）：[`PRD.md`](PRD.md)。

---

## 二、你要做什么

为 `manifest.json` 里列出的 **57 个动作**，每个生成 `frames` 张图，输出到：

```
assets/anim/<动作id>/frame_00.png, frame_01.png, ...
```

每张图的硬规格（详见 ART-PRD §3）：
- **512×512**、**PNG**、**透明背景**（RGBA）
- 全身居中，角色高度约占画布 80%
- **同一动作的所有帧：角色缩放/中心/脚底基线必须一致**（只有姿态变化）——这是动画不"抖"的命门
- 必须是 §2 那只狗（椒盐毛色、雪纳瑞眉毛胡子、蓝牛仔马甲、粉项圈、白色贴纸描边）

---

## 三、怎么做（关键：保证"每帧都是同一只狗"）

1. **传参考图，别凭空画**：每次生成都把对应参考图（见 `references/README.md`）作为 image2 的 reference / image-to-image 输入。
2. **先定基线**：先只做 `idle` 和 `eat` 两组，用 `ref-00-master-sit.png` 起手。**做完先停下让人确认风格**，通过后再批量推进其余 55 个。
3. **组内连贯**：做某个动作的多帧时，用该动作的第 1 帧做参考，保证 6 帧像同一个动作的连续瞬间。
4. **拼提示词**：每帧最终 prompt = `ART-PRD §3.2 风格前缀` + `§7 里该帧的动作描述` + `§3.3 技术后缀`（+ §3.4 负面提示）。
5. **优先级**：先做 Tier 1（22 个核心），再 Tier 2（20 个），最后 Tier 3（15 个彩蛋）。

---

## 四、做完怎么自检

每完成一批，跑：

```bash
npm run check:assets
```

它会对照 manifest 检查每个动作：
- `✅ idle (待机呼吸, 6帧)` = 合格
- `❌ eat: 缺 frame_03.png` = 缺帧/尺寸错/非PNG
- `⬜ dance — 未开始`
- 末尾给总进度，例如 `28/57`

**目标：全部 ✅，进度 57/57，退出码 0。**

---

## 五、边做边能看效果

代码已支持"缺帧自动回退"：没生成的动作会临时用旧图顶替，**App 始终能跑**。
你每填好一个文件夹，那个动作就自动变高清。随时可以启动预览：

```bash
npm start
```

操作：双击宠物开状态面板；点头/脸/肚子/爪/尾有不同反应；右键退出。

## 六、不要做的事

- ❌ 不要改 `manifest.json` 的 id / 帧数（改了代码和校验就对不上）。
- ❌ 不要改 `src/` 里的代码（渲染、交互、状态系统已完成；接图工作由项目方做）。
- ❌ 不要画"左"朝向版本（走路左向由代码镜像，见 ART-PRD §8.3）。
- ❌ 不要画带背景/地面/写实风的图。
- ❌ 别一口气出 57 个就交——**先 idle+eat 定基线确认**。
